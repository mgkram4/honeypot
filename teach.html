Introduction to Cybersecurity & Honeypots

What is Cybersecurity?

Cybersecurity is the practice of protecting computers, servers, mobile devices, electronic systems, networks, and data from digital attacks. These attacks typically aim to access, change, or destroy sensitive information, extort money from users, or interrupt normal business processes.

In today's digital world, organizations face constant threats from hackers, malware, and various network infiltration attempts. Cybersecurity professionals need tools to detect, analyze, and respond to these threats. One such powerful tool is a honeypot.

What is a Honeypot?

A honeypot is a cybersecurity mechanism designed to look like a legitimate system or network, but actually serves as a trap to detect, deflect, or study hacking attempts. Think of it as a decoy that attracts attackers, allowing security professionals to:

  • Study attacker tactics and techniques
  • Identify new types of attacks and vulnerabilities
  • Gather intelligence on threat actors
  • Divert attackers from actual systems

This tutorial will guide you through building a sophisticated honeypot system that uses artificial intelligence to detect and classify threats. Even if you're new to cybersecurity, we'll explain each concept thoroughly as we go.

Project Overview

The Honeypot AI Project creates a system that monitors multiple network ports (connection points on a computer) for potential attacks, analyzes these threats using machine learning, and displays the results on a real-time web dashboard.

Network Ports Explained

Network ports are virtual endpoints where network connections start and end. Think of them like different doors into your computer, each dedicated to a specific type of communication. Each port is identified by a number, and specific services typically use specific port numbers:

Port Service Description
21 FTP File Transfer Protocol - used for transferring files between computers
22 SSH Secure Shell - used for secure remote login to systems
23 Telnet Older, unsecured remote login protocol
80 HTTP Web browsing - unsecured website connections
443 HTTPS Secure web browsing - encrypted website connections
3306 MySQL Database communication

Our honeypot will monitor all these ports, which are common targets for attackers. When someone tries to connect to any of these ports, our system will collect information about the attempt and analyze it.

Key Features of Our Honeypot

  • Multi-port monitoring: Watches several network services simultaneously (FTP, SSH, Telnet, HTTP, HTTPS, MySQL)
  • Machine learning-based threat detection: Uses artificial intelligence to analyze and classify attacks based on patterns
  • Real-time web dashboard: Provides a visual interface to monitor attacks as they happen
  • Attack logging and persistence: Saves all attack data for future analysis and learning
  • Visual analytics: Displays threat patterns in easy-to-understand charts and graphs

Technical Components Explained

1. Honeypot Server (main.py)

The honeypot server is the foundation of our system. It creates "fake" services that look like real ones to attackers, but actually just record information about connection attempts.

How a Honeypot Server Works

When a computer runs services like FTP or SSH, it "listens" on specific network ports for incoming connections. Our honeypot does the same, but instead of providing the actual service, it just pretends to be that service. When attackers try to connect:

  1. The honeypot accepts the connection
  2. It records information about the connection (IP address, time, data sent)
  3. It may respond with fake information to make the attacker think they're interacting with a real service
  4. All of this happens without putting real systems at risk

Our honeypot server implements:

  • HoneyPot Class: The main class that handles network connections
  • Multi-threading: Uses separate processing threads to monitor multiple ports simultaneously (like having multiple security guards watching different doors)
  • Data capture: Records detailed information about each connection attempt
  • ML integration: Sends attack data to the machine learning module for analysis
  • Data persistence: Saves all attack information to JSON files (a structured data format) for later analysis
  • REST API: Provides a way for the web dashboard to request and receive data

2. Machine Learning Detection (ml/detection.py)

Machine learning allows our system to automatically identify patterns in attack data and classify threats without explicit programming for each attack type. This makes the system more adaptable to new threats.

What is Machine Learning?

Machine learning is a type of artificial intelligence that enables computers to learn from data and make decisions or predictions without being explicitly programmed for each scenario. In cybersecurity, it helps identify patterns and anomalies that might indicate an attack.

Our machine learning component:

  • Uses Random Forest Classifier: A powerful machine learning algorithm that works by creating many "decision trees" and combining their results
  • Extracts features from attacks: Converts raw attack data into numerical values the algorithm can analyze, including:
    • Time-based features: When the attack occurred (attackers often work at specific times)
    • Port-based features: Which service was targeted (indicates attacker's interests)
    • Content analysis: Looks for suspicious keywords like 'password', 'admin', 'root'
    • Data length analysis: Unusual data lengths might indicate exploit attempts

Our system classifies threats into three levels:

Low: Likely routine scanning

Most systems on the internet are constantly being scanned by automated tools. These low-level threats are typically just reconnaissance, where attackers are looking for systems that exist but aren't yet trying to break in.

Medium: Suspicious activity

This represents behavior that goes beyond simple scanning and shows intent to probe for vulnerabilities. The attacker might be testing different usernames or looking for specific software versions.

High: Probable attack attempt

This indicates an active attempt to exploit a vulnerability or break into the system. Examples include password guessing attacks, SQL injection attempts, or exploitation of known vulnerabilities.

What is a Web Dashboard?

A web dashboard is an interface that displays important information in an organized, visual way through a web browser. In cybersecurity, dashboards help security professionals monitor threats, visualize attack patterns, and respond quickly to incidents.

JavaScript (static/js/main.js)

JavaScript is the programming language that makes web pages interactive. In our dashboard, it:

  • Updates data in real-time: Fetches new attack information every 5 seconds
  • Creates visualizations: Uses Chart.js (a JavaScript library) to generate graphs and charts
  • Manages dynamic content: Updates the dashboard without needing to reload the page
  • Formats attack logs: Presents attack information in a readable format

Styling (static/style/style.css)

CSS (Cascading Style Sheets) controls how the web page looks. Our dashboard's styling includes:

  • Responsive grid layout: Adjusts to different screen sizes (desktop, tablet, mobile)
  • Threat level color coding: Visual indication of threat severity:
    • High threats: Red (#e74c3c)
    • Medium threats: Orange (#f39c12)
    • Low threats: Green (#2ecc71)

Setup and Installation

If you want to try running this honeypot system yourself, here's how to set it up:

Prerequisites

Before installing, you'll need:

  • Python installed on your computer (version 3.6 or higher)
  • Basic familiarity with running commands in a terminal or command prompt
  • Internet connection to download required packages
  1. First, install the required Python packages using pip (Python's package installer): 
    pip install flask scikit-learn pandas flask-cors

    This installs:

    • Flask: A web framework for creating the dashboard
    • scikit-learn: A machine learning library
    • pandas: A data analysis library
    • flask-cors: An extension for handling cross-origin requests
  2. Run the honeypot application: 
    python main.py

    This starts both the honeypot server (listening on the specified ports) and the web dashboard.

  3. Access the dashboard by opening a web browser and navigating to: 
    http://localhost:5000

    This opens the dashboard on your local machine. "localhost" refers to your own computer, and "5000" is the port the web server runs on.

Security Note: Running this system will open several ports on your computer. For learning purposes, it's best to run this on a system that's not exposed to the internet, like on your local network or personal computer.
